📘 Nexum Networking Documentation

    Overview


    Server-side send event handler of the Networking system.

    This event is responsible for transmitting encoded packets from the
    server to a specific connected client. The system validates the client
    identity before attempting to send the packet through the peer
    connection.

    If the target client is not registered in the server registry, the
    transmission is aborted and a diagnostic message is logged.

    LIBRARY:Call


    Handles the transmission of a packet to a client.

    The function retrieves the connection identifier from the peer object
    and checks whether the client exists in the server registry.

    If the client is not registered, the operation is aborted and an
    error message is printed indicating that the server attempted to
    send data to an unknown client.

    If the client is valid, the packet data is encoded using the server
    CODEC and sent through the peer connection using the specified
    channel and transmission flag.

    Parameters

    • table : Server instance
    • table : Event object containing peer, packet data, channel and transmission flag
